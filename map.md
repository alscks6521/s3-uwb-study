# UWB 실내 내비게이션 시스템 - 프로젝트 진행 정리

## 프로젝트 개요

실내 내비게이션 시스템 구축 프로젝트.
UWB(Ultra-Wideband) 실내에서 사용자의 정확한 위치와 방향을 파악하고, 음성으로 길을 안내하는 시스템.

---

## 사용 장비

| 구분 | 장비 | 수량 | 역할 |
|------|------|------|------|
| UWB 모듈 | MaUWB_DW3000 with ESP32S3 | 5개 | 앵커 4개 + 태그 1개 |
| 컴퓨터 | Odroid | 1대 | 중앙 처리 (위치 계산, 판단, 음성 출력) |


---

## 시스템 구성

### 앵커-태그 배분

- **앵커 (Anchor)**: A0, A1, A2, A3 → 사무실 직사각형 네 모서리 천장 근처(바닥에서 약 2.5m)에 설치
- **태그 (Tag)**: T0 (UWB_INDEX: 7) → 사용자가 휴대, 오드로이드에 연결

### 통신 구조

```
[A0] [A1] [A2] [A3]     ← 앵커 4개 (고정, 천장)
   \   |   |   /
    UWB 거리 측정 (TWR)
        |
      [T0]               ← 태그 (사용자 휴대)
        |
    WiFi UDP (포트 12345)
        |
    [PC / Odroid]         ← 위치 계산 + 시각화
    IP: 192.168.0.51
    WiFi: KIOT_Public_2.4G
```

## 사무실 측정 결과

### 측정 방법

줄자 대신 UWB 자체를 이용하여 측정.
T0를 각 앵커(A0~A3) 바로 밑으로 가져가서 나머지 앵커까지의 3D 거리를 수신한 뒤, 높이 성분을 보정하여 2D 수평 거리를 산출.

### 원시 데이터 (3D, 단위: cm)

| T0 위치 | → A0 | → A1 | → A2 | → A3 |
|---------|-------|-------|-------|-------|
| A0 밑 | 164 | 1719 | 1643 | 963 |
| A1 밑 | 1468 | 170 | 956 | 1760 |
| A2 밑 | 1690 | 964 | 258 | 1492 |
| A3 밑 | 948 | 1872 | 1504 | 173 |

> **참고**: A2 밑에서 자기 거리가 258cm으로 비정상적으로 높은 이유는 A2 부근에 단층(바닥 높이 차이)이 있기 때문.

### 높이 보정 (3D → 2D)

- 앵커 높이: 2.5m
- T0 높이 (손에 들고 있을 때): 약 0.85m
- 수직 거리: 약 1.65m (자기 앵커 밑에서의 측정값으로 확인)
- 보정 공식: `d_2D = √(d_3D² - h²)`

### 앵커 간 2D 수평 거리

| 구간 | 측정값 1 | 측정값 2 | 평균 | 신뢰도 |
|------|---------|---------|------|--------|
| A0-A3 (짧은쪽 하단) | 948.9cm | 932.1cm | **940.5cm** | ✓ 신뢰 |
| A1-A2 (짧은쪽 상단) | 940.8cm | 928.8cm | **934.8cm** | ✓ 신뢰 |
| A0-A1 (긴쪽 좌측) | 1711.2cm | 1458.1cm | 1584.6cm | ⚠ 오차큼 |
| A2-A3 (긴쪽 우측) | 1469.5cm | 1494.0cm | **1481.8cm** | ✓ 신뢰 |
| A0-A2 (대각선) | 1634.8cm | 1670.2cm | 1652.5cm | ✓ 신뢰 |
| A1-A3 (대각선) | 1751.8cm | 1864.0cm | 1807.9cm | ⚠ 오차큼 |

> A0-A1 좌측 벽 구간의 오차가 큰 이유는 벽이나 장애물로 인한 NLOS(비직선경로) 영향으로 추정. 직사각형 특성상 좌측=우측이므로 신뢰도 높은 우측(A2-A3: 14.8m)을 기준으로 채택.

### 최종 사무실 크기

**가로 9.4m × 세로 14.8m**

## 태그 펌웨어 (T0)

- **파일**: ESP32S3 Arduino 스케치
- **UWB_INDEX**: 7 (태그 ID)
- **PAN_INDEX**: 0 (앵커와 동일)
- **WiFi**: ..._Public_2.4G
- **UDP 전송 대상**: 192.168.0.51:12345
- **데이터 형식**: `AT+RANGE=tid:7,mask:0F,seq:N,range:(A0,A1,A2,A3,0,0,0,0),ancid:(0,1,2,3,-1,-1,-1,-1)`
- **거리 단위**: cm

### 데이터 예시

```
AT+RANGE=tid:7,mask:0F,seq:128,range:(997,1321,911,738,0,0,0,0),ancid:(0,1,2,3,-1,-1,-1,-1)
```

→ A0까지 997cm, A1까지 1321cm, A2까지 911cm, A3까지 738cm

## PC 시각화 스크립트 (uwb_map.py)

### 기능

1. UDP 포트 12345에서 T0 데이터 실시간 수신
2. AT+RANGE 데이터 파싱 (정규식)
3. 3D → 2D 높이 보정
4. 최소제곱법 삼각측량으로 (x, y) 좌표 계산
5. 이동 평균 필터 (5샘플) 적용
6. 두 가지 출력 모드:
   - **모드 1**: matplotlib 실시간 지도 시각화 (초록 점 + 이동 궤적)
   - **모드 2**: 콘솔 좌표 출력

### 실행 방법

```bash
pip install matplotlib
python uwb_map.py
```

### 검증 완료

- UDP 수신 ✅
- 데이터 파싱 ✅
- 삼각측량 좌표 계산 ✅
- 실시간 지도 시각화 ✅

## 기술적 참고사항

- **좌우 반전 문제 (Mirror Effect)**: 앵커 4개 배치로 해결됨
- **NLOS 오차**: 좌측 벽(A0-A1) 구간에서 발생. 필터링 또는 앵커 위치 미세 조정으로 보완 필요
- **A2 단층**: A2 부근 바닥 높이 차이 있음. 높이 보정 시 A2는 별도 처리 고려
- **DW3000 PDoA**: 위상차 기반 도래각 측정 지원 가능성 있음 (안테나 설계 의존)
